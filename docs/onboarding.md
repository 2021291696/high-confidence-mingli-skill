# Onboarding — 首次建档流程

> 触发：SKILL.md 启动检查发现 `~/.claude/fortune-telling/chart.md` 不存在。
> 目标：生成 chart.md（命盘数据）+ memory.md（运行时记忆），之后进入正常四模式路由。
> 原则：**先建档，后算命**——数据未就绪前不做任何运势判断。缺盘可降级，八字不可缺。

## 第 0 步：欢迎与隐私说明

向用户说明三件事：

1. 本 skill 是「三盘互译」命理引擎：八字 + 紫微斗数 + 七政四余互相验证，不给空泛断语
2. 建档需要出生信息；紫微/七政数据建议从排盘 App 粘贴（推荐：文墨天机、测测）
3. **所有数据只存在本地** `~/.claude/fortune-telling/`，不上传、不联网、不与任何账户绑定

## 第 1 步：采集基础信息

依次询问（一次问完，别挤牙膏）：

| 字段 | 必需性 | 说明 |
|------|--------|------|
| 称呼 | 必需 | 昵称即可 |
| 性别 | 必需 | 大运顺逆 + 十神感情映射（男看财/女看官）必需 |
| 公历出生日期 | 必需 | YYYY-MM-DD；农历生日请用户先换算或告知后由 agent 换算并确认 |
| 出生时间 | 必需 | 精确到分钟最佳；不知道 → 见下方「时间未知处理」 |
| 出生地点 | 必需 | 城市级即可，用于真太阳时校正 |

**时间未知处理**：只知大概时辰 → 按该时辰中点建档，chart.md 时柱标注「置信度：时辰估计」；完全不知道 → 三柱建档（年月日），时柱留空，所有涉及时柱的判断（子女宫、晚年运、部分格局判定）标注「时柱缺失」。

## 第 2 步：真太阳时校正

1. 查出生地经度（agent 内部知识，城市级精度足够）
2. 经度校正：`(当地经度 - 120°E) × 4 分钟`。例：成都 104°E → 慢 64 分钟
3. 均时差校正：±15 分钟内，按出生日期估算（agent 内部知识；不确定就标注「均时差未校正」）
4. 校正结果跨越时辰边界（如 21:05 → 20:48，从亥时跌入戌时）→ 必须告知用户并确认用哪个时辰

## 第 3 步：排八字四柱

按 SKILL.md §7 规则计算：

1. **年柱**：以立春为界（§6 预计算表；2004 年前出生者用 60 甲子回推：已知 2024=甲辰，每年回退一位）
2. **月柱**：节气为界 + 年上起月法。节气日期用 §6 表或 agent 历法知识，边界日（节气当天）标注
3. **日柱**：§7 锚点公式 `(11 + 天数差) % 60`，主锚点 2026-01-01 = 乙亥日
   - **强制交叉验证**：环境允许则 `pip install lunar-python` 离线验证；不允许则用验证锚点 2026-01-31 = 乙巳日自检公式；两者都不可行 → chart.md 标注「日柱未交叉验证」并请用户用排盘 App 核对
4. **时柱**：日上起时法（§7）
5. **十神 + 藏干**：§7 判定规则 + 藏干表
6. **格局/调候/强弱**：agent 按传统子平法分析，输出时附一句推理依据

排完后**展示完整四柱表请用户确认**，有排盘 App 的用户可当场核对。

## 第 4 步：紫微斗数（可跳过）

询问用户是否愿意提供紫微盘：

- **愿意** → 请用户从文墨天机/测测等 App 复制十二宫信息，按 `templates/chart.template.md` 的宫位格式粘贴。agent 整理为标准格式。最少需要：命宫主星+亮度、身宫位置、生年四化；十二宫全给最佳
- **不愿意/没有** → chart.md 标注「紫微数据缺失」，按缺盘降级规则运行（上限两盘同指）

⚠️ 本引擎**不做安星计算**（安星规则复杂易错，违反准确性原则）。紫微数据只接受用户导入。

## 第 5 步：七政四余（可跳过）

同第 4 步逻辑。最少需要：命度（星宿+五行）、躔、昼生/夜生、立命宫。

- 用户可提供七政盘截图文字或 App 导出
- 无法提供 → 标注缺失，按降级规则运行
- 用户只知道出生信息但想要七政 → 告知七政需要星历推算，本引擎不做，建议用专业七政排盘工具后导入

## 第 6 步：排大运

规则（阳男阴女顺排，阴男阳女逆排）：

1. 年干阴阳：甲丙戊庚壬=阳，乙丁己辛癸=阴
2. 阳年男 / 阴年女 → 从月柱顺排；阴年男 / 阳年女 → 从月柱逆排
3. 起运年龄：顺排数到下一个节气 / 逆排数到上一个节气，天数 ÷ 3 = 起运岁数（余数 1 天 = 4 个月）
4. 每步大运 10 年，生成大运表（干支 + 十神 + 起止年份），当前大运加粗
5. 计算边界模糊时标注，并请用户用 App 核对起运年龄

## 第 7 步：推导解释器种子

按 SKILL.md §3「推导规则」表，从已排盘面提取：

- 认知模式（印重/食伤旺/…）
- 情绪底色（太阴/太阳/昼夜生/…）
- 核心矛盾（原局最结构性的一组张力）
- 当前发展阶段（当前大运 + 时代运）

## 第 8 步：写盘并确认

1. 创建目录 `~/.claude/fortune-telling/`（不存在则建，含 `memory-archive/`）
2. 按 `templates/chart.template.md` 填数据 → 写 `chart.md`
3. 按 `templates/memory.template.md` 填种子值 → 写 `memory.md`
4. 向用户展示 chart.md 摘要（四柱 + 格局 + 已导入的盘 + 缺失的盘），请确认
5. 用户确认 → 告知「建档完成，可以问运势、推演问题、或聊聊命理概念」

## 维护操作（非首次）

| 用户说 | 操作 |
|--------|------|
| "重新建档 / 重置命盘" | 备份现有 chart.md 到 memory-archive/ 后重走本流程 |
| "补充紫微/七政数据" | 直接走第 4/5 步，更新 chart.md |
| "修改出生信息" | 确认后更新 chart.md 对应字段，重新推导受影响部分（时柱变→格局/解释器重推） |
