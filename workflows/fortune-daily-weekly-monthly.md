# Fortune-Daily-Weekly-Monthly — 运势查询 Workflow

> 由 SKILL.md §0 路由触发。处理今日/本周/本月/今年/指定月运势查询。
> 输入: query_type, query_date, domain(可选)
> 数据: `~/.claude/fortune-telling/chart.md` + `~/.claude/fortune-telling/memory.md`

## Phase 1: 加载

输入:
- `query_type`: 今日|本周|本月|今年|指定月|回顾
- `query_date`: 查询日期 (YYYY-MM-DD)
- `domain`: 可选，感情|事业|财运|健康
- 运行时画像: 从 memory.md 读取 (认知模式+情绪底色+核心矛盾+当前关注+已验证解释偏移)

执行:
1. 确定流年干支: 查 SKILL.md §6 预计算表，以立春为界
2. 推算流月干支: 用 §7 年上起月法，以节气日期为界
3. 推算流日干支: 用 §7 日干支推算规则（主锚点 **2026-01-01 = 乙亥日，0-based 序号 11**；公式 `(11 + 天数差) % 60`）
   - ⚠️ 历史 bug 教训：旧版曾误写锚点为「2026-01-01 = 乙巳日」，导致全日干支错位。**每次计算后必须用验证锚点 2026-01-31 = 乙巳日（序号 41）自检**；环境允许时用 lunar-python 离线交叉验证，无法验证则在输出中标注
4. 查流年四化: 用 §7 年干四化表
5. 计算关键冲合: 原局冲合 + 流年冲合
6. 边界检查:
   - 查询 2036+ → 标注「预计算表外，置信度降一级」
   - 节气边界当日 → 以 0 点是否已过该节气为准
   - 大运切换年 → 同时列出两运特征
   - 无法确定日期 → 返回「请问您想查询哪一天的运势？」

输出: { year_ganZhi, month_ganZhi, day_ganZhi, year_siHua, key_chongHe, current_daYun, boundary_notes }

## Phase 2: 并行信号提取

有子 agent 工具时 parallel（无则顺序内联执行，schema 不变）:

- **八字**: 提取该时段八字信号
  - 输入: 原局八字(chart.md) + 流年/流月/流日干支 + 冲合
  - 任务: 分析干支互动、十神变化、冲合引动
  - 重点: 日主与流年流月的关系、原局核心冲合是否被引动、印星/用神状态
  - schema: `{ signals: [{ element: string, direction: string, strength: "核心"|"次级"|"边缘", domain: string }] }`

- **紫微**（chart.md 含紫微数据时执行，否则跳过并记录缺盘）: 提取该时段紫微信号
  - 输入: 原局紫微(chart.md) + 流年宫位(§7公式) + 四化飞星
  - 任务: 分析流年宫位变化、四化飞星对原局的影响、神煞叠加
  - 重点: 流年命宫位置、生年四化与流年四化的交互
  - schema: 同上

- **七政**（chart.md 含七政数据时执行，否则跳过并记录缺盘）: 提取该时段七政信号
  - 输入: 原局七政(chart.md) + 当前大运五行 + 时代运五行
  - 任务: 分析躔度变化、相位关系、时代运对命度的作用
  - 重点: 命度状态、难仇恩用关系、时代运补/克效应
  - schema: 同上

## Phase 3: 综合分析

单线执行:

1. **置信度验证**: 对每组多盘关联跑 SKILL.md §10 三道题
   - Q1: 说的是同一个领域吗？
   - Q2: 能量方向一致吗？
   - Q3: 同名检测——同一能量的不同表达还是碰巧相似？
   - 判定: 三道全过→⭐⭐⭐ / 过前两道→⭐⭐ / 只过第一道→⭐ / 都不过→不提
   - 缺盘时按 SKILL.md「缺盘降级规则」封顶

2. **权重合成**: 按 §8 规则
   - 命格 40% 不参与截断重分配，始终以背景形式出现
   - 动态层按原比例重分配到 100%
   - 例: 今日运势展开流年(15%)+流月(7%)+流日(3%)=25% → 重分配: 流年60%/流月28%/流日12%

3. **领域筛选**（信号映射见 SKILL.md §13）:
   - 用户指定领域 → 主展开该领域，其他收缩为一句
   - 未指定 → 优先展开当前时段最活跃的领域
   - 感情和事业同时活跃 → 优先事业

4. **核心张力提取**: 找出最主要的能量信号 + 斯多葛视角

5. **练习建议**: 必须具体到当前能量状态，不是鸡汤

输出: { background, energy_analysis, core_tension, practice_advice, confidence_notes }

## Phase 4: 输出格式化

按 SKILL.md §9 斯多葛语气规则:

**禁用词替换**: 吉/凶/好/坏 → 能量状态描述; 你应该 → 你可以选择; 注意/小心 → 值得留意

**查询深度映射**:
| 查询类型 | 命格展开 | 流年 | 流月 | 流日 |
|---------|---------|------|------|------|
| 今日 | 一句 | 一句 | 概要 | 纹理 |
| 本周 | 一句 | 一句 | 概要 | 日柱走势 |
| 本月 | 一段(3-4句) | 简述 | 详解 | — |
| 今年 | 完整展开 | 详解 | 详解 | — |

**输出模板**:
```
## [查询类型]运势

### 背景
[命格定调，长度按查询深度缩放]

### [时间]能量
[流月/流日干支 + 关键冲合 + 多盘信号 + 置信度标注]

### 核心张力
[最主要能量信号 + 斯多葛视角]

> 🏛 你可以把握的是：[具体到当前能量状态的最小动作]
```

## 约束

- 回顾模式: 同查询逻辑 + 标注「回顾模式——只分析，不预测」
- 模糊查询("最近"): 默认展开到流月
- 意图识别("今天适合做X吗"): 映射到最匹配深度 + 针对性回答
- 用户沉默不反馈 → 不做推断，跳过校准
