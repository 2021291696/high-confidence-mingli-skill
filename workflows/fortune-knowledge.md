# Fortune-Knowledge — 通用知识 Workflow

> 由 SKILL.md §0 路由触发。处理脱离个人盘面的命理知识问答。
> 输入: concept
> 数据: SKILL.md 通用规则（§6/§7/§13）+ 可选对照 `~/.claude/fortune-telling/chart.md`

## 与共鸣溯源的边界

| | 共鸣溯源 | 通用知识(本workflow) |
|---|---|---|
| 用户绑定 | "我是不是……" | 无第一人称 |
| 分析锚点 | 个人命盘 | 通用命理体系 |
| 输出方向 | 溯源解释 | 知识解答 |

模糊边界: "偏印是什么？我是不是？" → 先本 workflow 解释概念，再走共鸣溯源对照，两段合一输出。

## Phase 1: 知识检索

从以下数据源提取与该概念相关的定义、规则、对照关系:
- SKILL.md §7 推算规则: 四化表、年上起月法、十神判定、藏干表等通用规则
- SKILL.md §13 领域信号映射: 该概念在各领域中的对应
- chart.md（仅用于可选对照，不作分析锚点）: 该概念在用户盘中的实例
- AI 自身命理知识: 概念定义、学派分歧、常见误解

输出: { definitions: string[], rules: string[], cross_references: string[] }

## Phase 2: 三盘对照

有子 agent 工具时 parallel（无则顺序内联）:
- **八字**: 该概念在八字体系中的表现
  - 任务: 定义/特征/常见误解
  - schema: `{ plate: "八字", manifestation: string, key_features: string[], has_direct_match: boolean }`

- **紫微**: 该概念在紫微体系中的表现
  - 任务: 定义/特征/常见误解
  - schema: 同上

- **七政**: 该概念在七政体系中的表现
  - 任务: 定义/特征/常见误解
  - schema: 同上

无直接对应的盘标注: "该概念在X盘中无直接对应"

## Phase 3: 输出格式化

按 SKILL.md §16 模板 + §9 斯多葛语气:

```
### [概念]是什么
[体系归属（八字/紫微/七政）+ 核心定义 + 常见误解澄清]

### 在各盘中的表现
| 盘 | 如何体现 | 关键特征 |
|----|---------|---------|
| 八字 | ... | ... |
| 紫微 | ... | ... |
| 七政 | ... | ... |

> 🏛 对照你的盘面：[有对应就对照，无对应就说明"无直接对应，但理解它有助于你更完整地认识命理体系"]
```

## 约束

- 不以个人盘为分析锚点（区别于共鸣溯源）
- 个人盘对照是可选附加，不强加
- 无第一人称绑定
