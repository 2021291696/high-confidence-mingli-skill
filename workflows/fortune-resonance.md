# Fortune-Resonance — 共鸣溯源 Workflow

> 由 SKILL.md §0 路由触发。处理外部内容(刷到的帖子/听说的概念)的命盘溯源。
> 输入: external_content
> 数据: `~/.claude/fortune-telling/chart.md` + `~/.claude/fortune-telling/memory.md`

## Phase 1: 内容解构

将用户引用的外部内容拆为可验证命题:
- **核心命题**: 内容主干主张（必须可验证）
- **边缘命题**: 附属描述（可选验证）

示例:
```
用户: "刷到说偏印人容易过度内化，做事慢半拍但想得深，我就是这样"

→ 核心命题A: 命盘中有偏印
→ 核心命题B: 偏印在命盘中起主导作用（印重/偏印格）
→ 核心命题C: "先内化再输出"的行为模式有盘面支撑
→ 边缘命题D: 这个模式在多盘中共振
→ 边缘命题E: 这个模式在当前大运/流年中被放大还是被抵消
```

输出: { propositions: [{ content: string, type: "核心"|"边缘" }] }

## Phase 2: 逐命题并行验证

对每个命题执行（有子 agent 工具时 parallel，无则顺序内联）:
- **八字验证**: 该命题在八字中的支撑/矛盾
  - 输入: 命题 + 原局八字(chart.md)
  - 仅使用八字信号，不引入其他体系
  - schema: `{ proposition: string, support: string, confidence: "✅✅✅"|"✅✅"|"✅"|"⚠️"|"❌" }`

- **紫微验证**（有数据时）: 该命题在紫微中的支撑/矛盾
  - 输入: 命题 + 原局紫微(chart.md)
  - 仅使用紫微信号
  - schema: 同上

- **七政验证**（有数据时）: 该命题在七政中的支撑/矛盾
  - 输入: 命题 + 原局七政(chart.md)
  - 仅使用七政信号
  - schema: 同上

**置信度标识**:
| 标识 | 含义 |
|------|------|
| ✅✅✅ | 三盘同指（三道题全过）——完全共振 |
| ✅✅ | 两盘同指（过前两道）——强相关 |
| ✅ | 单盘指向（仅过第一道）——部分相关 |
| ⚠️ | 仅命名相似——伪共鸣 |
| ❌ | 盘面无信号——无关 |

## Phase 3: 共鸣定性

1. **汇总所有命题验证结果**
2. **定性共鸣等级**:
   - 高度共鸣: 核心命题多盘同指 + 边缘命题 ≥ 50% 命中
   - 中度共鸣: 核心命题 ≥ 1 个两盘同指
   - 弱共鸣: 仅边缘命题命中，或全部仅单盘指向
   - 无共鸣: 全部 ⚠️/❌
3. **边界保护**:
   - 至少一个命题 ✅✅ 以上才说"有共鸣"（硬性门槛）
   - 所有命题 ⚠️/❌ → 直接输出"不匹配"，不硬凑
   - 禁止引入三盘外分析框架
4. **标注不完全匹配的部分**

输出: { resonance_level: "高度"|"中度"|"弱"|"无", matched_propositions: string[], unmatched_parts: string[] }

## Phase 4: 输出格式化

按 SKILL.md §15 模板 + §9 斯多葛语气:

```
### 你说的内容是
[一句话概括外部内容的核心主张]

### 你的盘面溯源
| 命题 | 八字 | 紫微 | 七政 | 置信度 |
|------|------|------|------|--------|
| 命题A | [信号] | [信号] | [信号] | ✅✅✅ |
| ... | ... | ... | ... | ... |

### 综合共鸣
[定性等级 + 说明：哪些命中是实质性的，哪些只是名字像]

### 为什么你会被这个内容击中
[从盘面结构解释共鸣根源——从能量配置讲清"不是巧合"，不说玄学腔]

### 不完全匹配的部分
[盘面数据不支持/矛盾之处——这一节是避免硬凑的关键]

> 🏛 共鸣不意味着你就是这个标签。你的盘比任何单一标签都复杂——这个内容描述了你[某个侧面]，但不是全貌。
```

## 约束

- 所有验证仅使用 chart.md 中已有的盘信号，禁止引入其他框架
- 用户沉默不反馈 → 不做推断
- 共鸣结果需写入 memory.md "共鸣档案"（由收束 workflow 处理）
